## Data engine (Python, scripts/)
- Archive: **~17.1M observations, Sep 2009 → present**, gaps down to **63 days**
  (~99% coverage). Remainder = station-dark periods. **May 2021 gap = confirmed
  WU platform data loss; do NOT re-hunt (ARCHITECTURE.md §5).**
- Sources + priority resolver in scripts/wxdb.py (schema authority): cumulus_log,
  mdb, amb_rest, amb_ws, wu_pws (KCALAJOL6 same roof), neighbor fills
  wu_ksandi354/4366/84 (~1-1.5 km). KCASANDI354 is the standing current-era
  backup for internet-outage gaps.
- Idempotent importers; canonical_observations view; daily_rollups (95k rows) is
  the workhorse for Records/Synopsis/Extremes. Rebuild: `python scripts/build_rollups.py`.
- Validation: `python scripts/validate.py` → data/DATA_REPORT.md.
- Quirks: MDB metric ×10 (solar lux ×10 ÷126.7→W/m²; UV µW/cm² ÷450→index);
  WU pre-2019 ms epochs; WU qcStatus 0 = skip; Ambient REST ≤1 req/sec.

## App (soledadwx-ui/)
- Left nav rail, 11 surfaces. **Built:** Console (MXUI-style dashboard, SteelSeries
  gauges, wind rose), Live (firehose), Min/Max Records (Cumulus-validated),
  "Analyze & Compare" (= Analyst-v1 plotter stand-in, really proto-Charts).
- Footer: station id, recording freshness, archive span, backfill indicator.
- Rust backend (src-tauri/src/): db.rs (recorder + status + records + backfill),
  lib.rs (WebSocket + commands). db.rs mirrors wxdb.py schema.
- **Auto-backfill-on-launch** (db::backfill, reqwest): on open, pulls Ambient
  cloud from last-recorded to now (amb_rest). Handles "PC was off." Neighbor-fill
  handles "internet was down."
- Backfill/recorder write observations only → daily_rollups need a Python rebuild
  for Records/Synopsis to reflect new live data.
